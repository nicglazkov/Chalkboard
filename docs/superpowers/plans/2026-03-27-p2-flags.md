# P2 Flags: Audience, Tone, and Theme

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add three new CLI flags — `--audience`, `--tone`, and `--theme` — that flow through `PipelineState` and shape the AI-generated script and Manim animation.

**Architecture:** Each flag follows the same pattern: new `Literal` field in `PipelineState` → default in `config.py` → CLI arg in `main.py` → passed through `run()` → picked up by the relevant agent via `state.get("field", "default")`. Audience and tone both inject a one-line instruction into `script_agent`'s user message. Theme injects color constants into `manim_agent`'s user message using a `THEME_SPECS` lookup dict. All three fields are added to `_init_state` in `graph.py` so they survive LangGraph's state merge.

**Tech Stack:** Python stdlib only; Anthropic SDK (existing); no new dependencies.

```python
def _build_user_message(state: PipelineState) -> str:
    topic = state["topic"]
    effort = state["effort_level"]
    feedback = state.get("fact_feedback")
    web_approved = state.get("user_approved_search", False)

    msg = f"Topic: {topic}\nEffort level: {effort}"
    msg += f"\n{AUDIENCE_INSTRUCTIONS[state.get('audience', 'intermediate')]}"
    if feedback:
        msg += f"\n\nPrevious attempt had issues. Please rewrite the script fully, addressing this feedback:\n{feedback}"
    if web_approved:
        msg += "\n\nWeb search has been approved — use it if needed."
    if effort == "low":
        msg += "\n\nEffort=low: keep the script concise, 3–4 segments, no web search needed."
    return msg
```
